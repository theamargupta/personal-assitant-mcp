### Step 2 — `.claude/commands/`

**`add-mcp-tool.md`** — frontmatter `description: Scaffold a new MCP tool`; body instructs to pick the right file under `lib/mcp/tools/` by domain (habits/tasks/documents/finance/goals/memory), register in `lib/mcp/server.ts`, Zod v4 input, IST timestamps via `types/index.ts` helpers, log to appropriate access log table. Takes `$ARGUMENTS` for tool name/description.

**`add-migration.md`** — next NNN after `008_memory_hybrid_search.sql`. RLS required. For memory tables use `pa_` prefix. Embedding columns 1536-dim.

**`add-widget.md`** — scaffold a new HTML file under `widgets/` and update the postinstall bundler script reference. Remind user the tool must use `registerAppTool` with `_meta.ui.resourceUri`.

**`ist-timestamp-check.md`** — grep for `new Date()` and `Date.now()` across `lib/` + `app/api/`. Flag any that don't flow through IST helpers in `types/index.ts`. Report only.

**`run-tests.md`** — run `npm run type-check`, `npm run test`, `npm run lint`. Report per-command status.

**`memory-consolidate.md`** — output SQL to inspect duplicate candidates (cosine ≥ 0.9 via `pa_match_memories`) and stale items across `pa_memory_items`. Do not execute.

### Step 3 — `.claude/agents/`

**`mcp-tool-reviewer.md`** — audit every file in `lib/mcp/tools/`:
- Zod v4 inputs
- OAuth user id extracted from session, not body
- Service-role client for DB ops
- Access log writes per call
- IST for any date math
- Soft delete only (`is_active=false, invalid_at=now()`)

**`oauth-reviewer.md`** — PKCE S256, SHA-256 hashed tokens, validated redirect_uri, single-use auth codes, revocation works.

**`migration-reviewer.md`** — NNN sequence, RLS on every new table, `pa_` prefix for memory, 1536-dim embeddings, FK indexes.

**`widget-reviewer.md`** — every widget HTML in `widgets/`:
- Sends `postMessage` to parent for all state
- Respects `prefers-color-scheme`
- No external script loads (self-contained bundle preferred — check `scripts/gen-ext-apps-bundle.mjs` for the bundle output location)
- Each widget has a matching MCP tool registered via `registerAppTool` with `_meta.ui.resourceUri`

**`ist-timezone-enforcer.md`** — grep for `toISOString`, `Date.now`, `new Date(`, timezone offsets. Ensure all date math uses IST helpers.

**`memory-dim-guard.md`** — ensure embedding vectors stay 1536-dim across `pa_memory_items`, `wallet_document_chunks`, and code. Flag mismatches.

### Step 4 — `.claude/hooks/`

**`block-env-commit.sh`** — standard.

**`block-hard-delete-memory.sh`** — PreToolUse for Write/Edit; flag `drop table pa_memory`, `delete from pa_memory_items`, etc. Require soft delete.

**`enforce-ist.sh`** — PreToolUse for Write/Edit on `.ts`/`.tsx` files under `lib/` and `app/api/`; if the new content contains `new Date()` without a nearby IST helper import, emit a warning (non-blocking — exit 0 with stderr note).

**`lint-on-write.sh`** — same pattern, runs eslint on `.ts`/`.tsx`.

**`prevent-tailwind-config.sh`** — standard.

```markdown
# API Conventions

## MCP (`api/mcp/route.ts`)

## REST (`api/finance/*`, `api/documents/*`, etc.)
- Zod v4 body validation.
- Supabase SSR client for user context; service-role for backend-only operations.
- Response: `{ success, data }` / `{ success, error }`.
- IST timestamps.
```

```markdown
# MCP Server
- `server.ts` factory returns a fresh server per request.
- Tools registered from `tools/{habits,tasks,documents,finance,goals,memory}.ts`.
- Memory tools with widgets use `registerAppTool({ _meta: { ui: { resourceUri } } })`.
- All inputs validated via Zod v4.
- Never hard delete.
```

**`lib/memory/CLAUDE.md`:**
```markdown
# Memory Vaults
- Tables prefixed `pa_` (coexist with memory-mcp on the same Supabase DB).
- Embeddings: 1536-dim.
- Duplicate detection: `pa_match_memories` at cosine similarity 0.9. `save_memory` returns `status: duplicate_candidate` unless `force: true`.
- Hybrid search: `pa_hybrid_search` (vector + tsvector). `search_vector` column maintained by trigger.
- Stale hints: `computeStaleHint(importance, valid_at, updated_at)` — server-side only.
- `consolidate_memories` returns duplicate groups and stale candidates.
```
