### Design Language (inherit from landing)
- Dark background `bg-primary`, translucent panel cards `bg-white/[0.01]`, sparse borders `border-white/[0.04]`.
- Accent: `text-neon` (lime-green from Tailwind tokens). Use sparingly — highlight one metric or one CTA per screen.
- Typography: big numbers on stat cards (`text-[32px] font-bold tracking-[-0.02em]`), muted labels above them (`text-[11px] uppercase tracking-[0.25em] text-text-muted`).
- Radii: `rounded-2xl` for cards, `rounded-full` for chips and pill buttons.
- Framer Motion on entrance: reuse the `container` + `item` stagger pattern from `components/landing/BentoShowcase.tsx`.
- Inspiration from Sathi's Android app (see `/public/landing/mobile-*.png` for reference): greeting hero, colored-dot list items, pill "Log" buttons, priority color chips (high = red-ish accent, medium = amber/text-secondary, low = muted), prominent ask-bar at bottom.

   - Visual: small colored dot (matches `accent`) + uppercase label, huge value, optional hint in `text-text-muted`. Subtle hover lift.

3. `components/dashboard/kit/SectionHeader.tsx` — section title with optional eyebrow + right-side action slot.

4. `components/dashboard/kit/Chip.tsx` — pill badge. Variants: `priority-high | priority-medium | priority-low | status-pending | status-in-progress | status-completed | tag`. Priority-high uses a soft red/neon-accented style; low is muted.

All kit files: `'use client'` if they use Framer Motion or hooks; otherwise server component by default. Export named exports.
