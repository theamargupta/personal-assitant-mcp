Every CRUD function must:
1. Call `createClient()` 
2. Call `supabase.auth.getUser()` and check user exists
3. Perform the Supabase query
4. Call `loadData()` to refresh the list
5. Show toast feedback on success/error

1. **Upload Document** — Add "Upload" button in the header. Opens modal with:
   - File input (accept: .pdf, .png, .jpg, .jpeg)
   - Name (text, defaults to filename)
   - Description (optional textarea)
   - Doc type (auto-detect from mime: pdf/image/other)
   - Tags (comma-separated text input)
   
   On submit:
   a. Insert row into `wallet_documents` with `status: 'pending'`, `user_id`, `name`, `description`, `doc_type`, `mime_type`, `file_size`, `tags`, generate a `storage_path` as `{user_id}/{uuid}_{filename}`
   b. Upload file to Supabase Storage bucket `documents` at that path using `supabase.storage.from('documents').upload(storagePath, file)`
   c. Update the document row: set `status: 'ready'` (skip the text extraction step — that's server-side only via MCP)
   d. Show toast "Document uploaded!"
   e. Refresh the list

2. **Delete Document** — Add a trash icon on each document card (visible on hover). Confirm dialog. Then:
   a. Delete from Supabase Storage: `supabase.storage.from('documents').remove([doc.storage_path])`
   b. Delete related chunks: `supabase.from('wallet_document_chunks').delete().eq('document_id', doc.id)`
   c. Delete document row: `supabase.from('wallet_documents').delete().eq('id', doc.id)`
   d. Toast + refresh

3. **Search enhancement** — The current search is client-side by name/tag. Keep it as is — semantic search requires server-side embeddings (MCP only). But add a note below the search input: "For content search, ask Claude" in text-text-muted.

## Technical Constraints

1. **Framework:** Next.js 16.2.3, React 19, TypeScript strict
2. **Supabase client:** Always use `import { createClient } from '@/lib/supabase/client'` — it returns a browser client with anon key
3. **Auth pattern:** Every function that touches the DB must call `const { data: { user } } = await supabase.auth.getUser()` and bail if no user
4. **RLS:** All tables have Row Level Security. The `user_id` column matches the authenticated user. Always include `user_id` in inserts.
5. **Animation:** Use `framer-motion` for modals (AnimatePresence) and list items (stagger). Already imported in all files.
6. **Styling:** Use the monochrome + neon lime design system described in the Color System section above. Replace ALL old accent-blue/glass tokens.
7. **Toast pattern:** `const [toast, setToast] = useState('')` + `setTimeout(() => setToast(''), 2000)` + fixed bottom-right div
8. **Supabase Storage bucket:** `documents` — upload with `supabase.storage.from('documents').upload(path, file)`
9. **Date handling:** Use `new Date().toISOString()` for timestamps, `.split('T')[0]` for date-only
