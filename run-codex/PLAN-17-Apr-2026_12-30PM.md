## 1. Migration — [supabase/migrations/009_task_categorization.sql](supabase/migrations/009_task_categorization.sql)

```sql
ALTER TABLE tasks
  ADD COLUMN task_type TEXT NOT NULL DEFAULT 'personal'
    CHECK (task_type IN ('personal', 'project'));

ALTER TABLE tasks
  ADD COLUMN project TEXT NULL;

ALTER TABLE tasks
  ADD CONSTRAINT tasks_project_required_for_project_type
  CHECK (task_type = 'personal' OR (task_type = 'project' AND project IS NOT NULL));

CREATE INDEX idx_tasks_user_type_project ON tasks (user_id, task_type, project);
```

- Existing rows: default `'personal'` + `project = NULL` satisfy the CHECK — no backfill needed.
- RLS on [tasks](supabase/migrations/001_habits_and_tasks.sql) already scopes by `user_id` — nothing to change.
- FK index rule (per [supabase/CLAUDE.md](supabase/CLAUDE.md)): no new FK, just the composite index above.

## 3. New MCP tool — `get_task`

**Location:** append to [lib/mcp/tools/tasks.ts](lib/mcp/tools/tasks.ts). Plain `server.tool()` (no widget).

**Input:** `{ task_id: z.string().uuid() }`

**Behavior:**

1. Service-role select task by `id + user_id` — 404 if missing.
2. Build base response: `{ task: { id, title, description, status, priority, due_date, tags, task_type, project, created_at, completed_at } }` (IST-formatted timestamps via `toIST`).
3. If `task.task_type !== 'project'` → return `{ task, project_context: null }`.
4. Else build `project_context`:
   - `summary`: SELECT from `pa_memory_items` where `user_id = ? AND project = ? AND is_active = true`; aggregate `total_memories` + `by_category` (group-by count).
   - `rules`: delegate to existing `getRules(userId, task.project)` in [lib/memory/items.ts](lib/memory/items.ts). Include full rows (id, title, content, tags).
   - `relevant`: call existing `searchMemories({ userId, query, project: task.project, limit: 10 })` where `query = [task.title, task.description].filter(Boolean).join(' — ').slice(0, 500)`. Return score breakdown like `search_memory` does.
   - `claude_md_hint`: static string — `"Load the project CLAUDE.md and any nested CLAUDE.md files touching this task. See project='${task.project}' rules above for hard constraints."`

**Registration:** add `// ── get_task ──` block; no change needed in [lib/mcp/server.ts](lib/mcp/server.ts) since `registerTaskTools` is already wired.
